neurotrace — Definition of Done

Locked. This file is not edited mid-loop. Scope changes go through the user, not silently. Every gate below must be checked by a context/agent that did not write the change (L4 VERIFY is always separate from the maker).

§1 — Cognitive job (Phase 0)

Written per the agentic-swe-master 5-question diagnostic, run against neurotrace as it exists today (v0.1.0, existing system being extended, not greenfield).

  • Scope: existing system being extended (not new, not an incident).
  • AI components: none directly — neurotrace makes no LLM calls itself. It instruments code that does. Its own "reasoning" is zero: capture, store, render. No autonomy.
  • Distribution: none — single process, local SQLite file, loopback-bound HTTP server. Not a distributed system.
  • Trust boundary: yes, and it's the sharpest edge in the project — traces store prompts/tool-args/results verbatim and unencrypted by design. The boundary is "this machine" (trusted) vs. "network / anyone reading the .db" (untrusted).
  • Cognitive job, stated: capture whatever an agent's LLM/tool calls produce, exactly as produced, without altering or short-circuiting the agent loop being observed; persist it; make it inspectable after the fact via CLI, JSON API, and browser UI. Human involvement is total — a person reads the trace; neurotrace never acts on what it captures.
  • Failure tolerance: the observer must never break the observed. A tracer or storage failure must degrade (partial trace, a warning) — it must not raise into the host agent's control flow.

§2 — Phase gates touched by this plan

Copied from agentic-swe-master's gate lines for the phases that actually apply (see wiki/index.md for why each does or doesn't). Only phases with in-scope work below.

Phase 10 — Observability & Tracing. Gate: every agent span captures latency + token count + tool result + cost. Alerts on p99 latency, error rate, cost drift. Prompt traces searchable.
Applicability here: latency/token/tool-result capture already holds for non-streaming LLM calls (pre-existing). Streaming spans do not meet this gate yet (M3). Cost-in-dollars and alerting are out of scope — no live system to alert on.
Phase 11 — Security Architecture. Gate: threat model exists with specific adversary categories. Prompt injection mitigated with instruction hierarchy and sanitization. RBAC implemented. Audit trails append-only, tamper-evident.
Applicability here: no RBAC/audit-trail requirement (single-user local tool). The relevant slice: sensitive data written verbatim by design, with no way to opt out today. M2 closes the "no way to opt out" gap. Full threat-model doc is out of scope for this pass — the invariant in context-graph.json stands in for it.
Phase 12 — Reliability Engineering. Gate: every outbound call has circuit breaker + timeout + retry with idempotency. Checkpointing allows resumption after crash. No operation has side effects on retry it didn't have on first execution.
Applicability here: "outbound call" here means the SQLite write, not a network call. Partial-trace-on-exception already holds for tracer-level failures; M4 extends it to storage-level failures and makes re-instrumenting a client idempotent.
Phase 18 — CI/CD for AI Systems. Gate: no change deploys without passing eval gates.
Applicability here: "eval gate" = the existing 58-test pytest suite + lint. Currently there is no automated gate at all (M1). No shadow/canary deploy concept applies to a pip package.

§3 — Milestone mirror (see PLAN.md for full detail)

M1 — CI/CD gate. Done when: .github/workflows/ci.yml exists, runs ruff check + pytest on push/PR, and a verifier confirms it actually ran green on the current branch (not just that the file exists).
M2 — Redaction hook. Done when: tests/test_redaction.py passes, default (no hook supplied) behavior is provably unchanged (existing storage tests still pass unmodified), and README's "Data handling" section is updated to describe the opt-in hook instead of only describing the gap.
M3 — Streaming traced. Done when: a streamed completion produces a span with non-empty assembled content, a recorded time-to-first-chunk, and a recorded total duration; the caller's chunk iteration is unaffected (existing streaming behavior, if any caller depends on it, does not regress); examples/streaming_agent.py demonstrates it end to end without an API key.
M4 — Storage failure isolation. Done when: tests/test_tracer_reliability.py proves a raising storage backend does not propagate into the traced code, a warning is emitted (not silently swallowed), and calling trace_openai twice on the same client does not double-record spans.

Verification rule

A milestone is not "done" because the maker's own tests pass. It is done when a separate verification pass (different context, ideally a different model) re-runs the demo command from PLAN.md from a clean checkout and confirms the gate text above, independent of the maker's summary of what it did.
